# 评审团协议(深档增强 · 显式点名才启动)

> 「评审团」= 把体检从"一个 AI 过一遍"升级成"**多位各守一个透镜的评委盲审 → 反驳者逐条对抗验证 → 主笔合成**"。
> 独立性是它的全部价值:评委互相看不见,反驳者只负责推翻。

## 何时用(闸门 · 先过这关)
- 只在用户**显式点名**(说"上评审团 / 评审团体检 / 多专家会诊")时启动,**绝不默认触发**。
- 启动前必告知成本:评审团会派出数十个子代理,消耗约为标准档的 **30–50 倍**;只值得用在正式交活/过评审的 key message、品牌手册级物料上,日常一条 message 速检/标准档足够。**按实际执行层级如实告知**——降级跑"多轮自审"时消耗远低于满配,别照念满配数字。
- **确认规则**:用户点名"上评审团"且材料已给全 = 视为确认,报一句成本即开跑,不再二次拦;点名但没给材料 → 先走首次引导收材料。

## 三级执行(按环境能力择一 · 诚实告知用户跑的是哪级)
1. **评审团**(首选):环境有多智能体编排工具(如 Claude Code 的 Workflow)→ 按本协议现场编排:评委并行盲审 → 合并去重 → 反驳者逐条投票 → 主笔合成。真并行、真盲审。
2. **并行子审**:环境只有并行子代理(如 Agent/Task 类工具)→ 每位评委/反驳者一个独立子代理,保住"盲审 + 独立反驳"实质;可把每条挑刺的反驳者从 4 位减到 2 位(**必留"行业惯例校准"**)。
3. **多轮自审**(兜底):单对话环境 → 逐透镜轮流批判(每轮只戴一个透镜,刻意不回看上一轮结论)→ 反驳轮 → 合成。**必须明确告知用户:这是评审团的单机版,后一轮看得见前一轮,独立性弱一档。** 单机版反驳轮可精简到 2 位(**必留"行业惯例校准"**);每轮重新读一遍该透镜的判据文件、禁止引用前轮条目编号——用行为约束近似盲审。

## 评委席(盲审 · 4–5 位 + 追问席)
| 评委 | 只看一件事 | 标准来源 |
|---|---|---|
| 大词虚词 | 逐句扫大词/虚词/万能话("卓越""领先""全面守护"),挑出换个产品名也成立的句子 | `大词虚词黑名单.md` |
| 有效信息三要素 | 每条信息实不实:有没有具体病人/具体场景/具体动作,支撑到位吗 | `有效信息三要素.md` |
| 策略可执行性 | 定位/策略推得出医生动作吗?战场选对没?主信息一条还是散的? | `策略可执行性检验.md` |
| 合规红线 | 编数据/超适应症暗示/绝对化宣称/贬低竞品越界/禁用词 | `合规红线.md` |
| 视觉信息权重(条件席) | **仅当输入是展架/海报/slide 等有版面物料时入席**:视觉权重与信息权重倒挂没? | `视觉信息权重倒挂.md` |

**追问席**(独立运行,不参与挑刺):扮演目标科室处方医生(**2 问**)与市场部上级(**1 问**),给出当面最会顶回来的问题(医生:"这跟我下一个病人有什么关系?我为什么信?";上级:"so what?这句话竞品说不了吗?"),合计 **3 个最狠问题**,必须具体到这份 message。

每位评委规则:
- 只从自己的透镜挑刺,**1–4 条最狠的,宁缺毋滥**;每条必引原句。
- 每条给:命中什么失败信号 → 为什么 → 严重度(高/中/低) → 可落地的改写建议(过有效信息三要素)。
- 各自独立给评级(不及格/及格/良好/优秀)+ 一句话理由。评委之间互相看不见。

## 合并去重
指向同一问题的挑刺合并成一条(保留最狠表述 + 最好的改写建议),记录"哪些评委命中"——**多透镜共同命中本身是强度信号**。不丢独立问题、不自己新增。

## 反驳席(对抗验证 · 每条合并后挑刺过 4 位反驳者)
| 反驳者 | 攻击角度 |
|---|---|
| 处方医生视角 | 站在真实处方医生角度,这条挑刺成立吗?还是评委想多了、医生根本不这么读? |
| 落地操盘手 | 写过上百条 key message 的老手:真问题还是吹毛求疵?改写建议在数据可得性/审批上落得了地吗? |
| 方法论核查 | 有没有误用标准(如拿深档全套标准轰一句随手发来的 message、把用户没提供的数据当"无支撑")?引用的原句真的在输入里吗? |
| 行业惯例校准 | 这在真实医药市场物料里是不是**普遍且被接受**的表达?别把行业常态当红线(合规硬红线除外——绝对化宣称/超适应症该拦必须拦)。 |

规则:每位反驳者的任务是**尽力推翻**这条挑刺,推翻不了才放行;模棱两可倾向推翻。**≥2 票推翻 = 毙掉**(进纪要);恰好 1 票反对 = 幸存,但反对意见作为"少数派意见"附在该条下。**例外(只限清单型)**:原句属实、且用词**确在红线清单上**(如绝对化宣称"卓越/最佳/根治")的合规挑刺不适用票决,直接保留(合规不搞少数服从多数);**判断型合规定性**(如"暗示超适应症获益"这类需要推断的)按普通挑刺走票决,可被反驳席降为"提示级风险"而非红线。

## 主笔合成(输出必须与标准档同构)
最终报告**落 SKILL.md 的统一输出模板五段式**(🔴卡在哪 → 🗣会被追问 → 🟢删/补/换 → 🎯证明 → 🚫失败信号),与速检/标准档同构、可截图沉淀。评级合议:取评委多数意见,平票取更保守(更低)的一档;**踩清单型合规红线 = 直接不及格**(一票否决,不合议)。

评审团特有内容作为**追加段**,不另起格式:
```
⚖️ 评审团纪要(追加段)
- 各评委评级 + 分歧点
- 每条挑刺标注:✓N/4 票幸存 · 提出者透镜 · 少数派意见(如有)
- 被毙掉的挑刺 + 推翻理由(评审团的自我纠错,照实写,不藏)
- 追问席 3 问(并入 🗣 会被追问 段)
```

诚实红线:0 条被毙也照实说,并提醒用户**人工抽查 2–3 条**——反驳席不是万能的,行业判断的最后一道校准永远在人。
