# 评分卡（及格 / 良好 / 优秀）

给"定位 + 关键信息"打分。先判合规，再判质量。

## 一票否决（直接不及格）

- 命中任一合规红线（绝对化宣称 / 裸数据 / 超适应症）。
- "换个产品名也成立"——即无任何差异化。

## 四个维度（各看是否达标）

| 维度 | 不达标 | 达标 |
|---|---|---|
| **战场清晰** | "成为领导品牌"等结果/目标 | 能还原"抢哪群病人·凭哪个优势"（见 策略可执行性检验.md） |
| **有主线** | 多个平级口号堆砌 | 1 主信息 + 支撑 |
| **信息够实** | 全是大词 | 每条过有效信息三要素 |
| **合规干净** | 有风险词/裸数据 | 0 风险、数据有出处占位 |

## 评级映射

| 评级 | 条件 |
|---|---|
| **不及格** | 触发一票否决，或 ≥3 个维度不达标 |
| **及格** | 有方向但仍大词为主；2 个维度不达标 |
| **良好** | 有主线、有具体病人；仅 1 个维度（多为支撑偏虚或数据缺出处）不达标 |
| **优秀** | 4 维度全达标：战场清晰 + 有主线 + 信息实 + 合规干净，且有"可带进诊室的一句话" |

## 输出格式（固定结构，便于沉淀/对比/当内容素材）

输出顶部先带一行**元数据**(便于归档、对比上次)：

```
[产品/适应症 · 体检日期 · 档位 · 评级]

评级：__（一句话总诊断）
🔴 挑刺：逐条 —— 每条必须「引用户原句」，格式：「原句」→ 问题 → 它其实想说什么
🟢 改写：主信息 + 支撑×2
🎯 深档：医生/上级当面会顶你的 1–3 个问题
```

**硬规则:每条挑刺必须先引用户的那一句原话**(如「'卓越疗效'这句」)，不准只笼统说"有大词/有问题"。指到原句，用户才知道改哪——这是"实用"和"正确的废话"的分界。
