# 测试用例(关键信息体检)· 脱敏

> 发布前必跑。每条 = 输入 + 客观验收点。有 skill vs 无 skill 对比("无 skill 组"显式禁用已装 skill)。改版后回归。

## T1 · 典型(全大词 message)
**输入**:「络平宁,新一代降压治疗选择,全面守护患者心血管健康。卓越疗效,值得信赖——医患共同的安心之选。」适应症:原发性高血压,受众:心内科医生。
**验收点**:评级=不及格;必须挑出 ①"卓越疗效"绝对化(合规红线) ②"全面守护心血管健康"超适应症暗示风险 ③换个产品名也成立(没定位);改写落地版过"有效信息三要素"(具体病人/可记住一句话/医生动作)。

## T2 · 边界:已经很好的 message
**输入**:「对单药治疗 8 周未达标的高危高血压患者,络平宁联合方案在 [RCT 名占位] 中示诊室收缩压较基线下降 [X]mmHg(vs 对照 [Y]),达标率 [Z]%——下一位复诊未达标的患者,可评估起始联合。」适应症:原发性高血压,受众:心内科医生。
**验收点**:评级=良好/优秀;**不硬挑刺**(占位符 [X] 不算"编数据"——那是本 skill 自己要求的占位规范);允许指 1–2 个真实提升点;不得堆砌批判。

## T3 · 边界:糙输入
**输入**:「帮我看看这句行不行:疗效好,安全性佳」。
**验收点**:走速检;只挑最狠 1–2 条(换名成立/无支撑);不跑全量模板;不脑补适应症,缺的信息问一次或标 `[待确认]`。

## T4 · 有版面的物料
**输入**:一页展架描述:「顶部大字:健康血压 美好生活(占 1/2 版面);中部:产品名+两行小字疗效数据;底部:公司 logo。」
**验收点**:加查"视觉信息权重倒挂"(情绪口号占大半版面、核心信息缩在小字);其余维度照常。

## T5 · 合规
**输入**:「络平宁,根治高血压的最佳选择,无任何副作用。」
**验收点**:三处全拦(根治/最佳=绝对化、无任何副作用=违背药品常识的安全性宣称);评级=不及格(红线一票否决);MA/RA 提示。

## T6 · 守窄边界(防越界代审)
**输入**:一份完整 BP 的"问题+策略页"(三条方向式策略)+ 请求「帮我体检这份 BP」。
**验收点**:**不整份接走**——说明本 skill 审的是其中"关键信息/策略表述那句话"的表达质量;策略推导扎不扎实、汇报结构好不好是另外的活,温和说明分工;可以就"策略那句话"的大词问题给速检级意见。**把整份 BP 按五维策略尺全审 = fail。**

## T7 · 评审团档回归(需多智能体编排环境;无则验证降级)
**输入**:T1 的 message + 点名"上评审团"。
**验收点**:①先报成本;②4–5 评委盲审(无版面物料时视觉权重席不入席)+ 清单型合规免票决;③落契约五段式+⚖️纪要;④无编排环境明说降级"多轮自审"。

- 无 skill 对比跑 T1(prompt 显式声明"不要使用任何已安装 skill")。
- 改 SKILL/references 后回归。渲染样例(全大词-不及格 / 优秀档-空挑刺 的 json+html)在本目录,供对比卡管线回归用。
