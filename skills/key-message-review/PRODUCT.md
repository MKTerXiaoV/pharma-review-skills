# Product — 关键信息体检（关键信息体检 skill）

这份文档讲**为什么**。事实在 `HANDOFF.md`。

## 1. 真正解决什么问题

**不是"帮市场部写文案"，是"市场部自己看不出自己的文案在说正确的废话"。**

医药市场部每周都在写医生材料、品牌 message、产品介绍。现实链路：
1. 写出来的 key message 大多是"全面提升 / 卓越疗效 / 创新引领 / 理想之选"。
2. 每个竞品都这么写，医生一条都不信。
3. 自己看不出问题——因为这些话"都对"。
4. 有的还埋着合规雷（绝对化宣称、裸数据、暗示扩适应症），自己没意识到。
5. 通用 AI 文案工具不懂治疗领域、更不懂处方药合规，只会把大词换成更花的大词。

我们插在第 2–5 步之间：**先体检挑刺，再改落地，最后写成医生愿读的材料，并自动拦合规雷。**

## 2. 定位（一句话）

> 把产品的基础信息交给它，几分钟内拿到一份"定位/关键信息体检报告 + 落地改写 + 一篇面向医生的产品优势介绍"，**有脾气、懂医药、自带合规红线**——而不是又一段正确的废话。

## 3. 护城河

通用工具做不了的三件事，是这个产品的命门：
- **懂判断**：能说出"你把最硬的机制故事讲虚了、把最不硬的疗效当主攻"。
- **懂合规**：自动拦绝对化宣称、裸数据、超适应症。
- **能落地**：改写出"医生明天能带进诊室的一句话"，不是口号。

生成谁都会，**审视 + 合规拦截 + 落地改写**是壁垒。

## 4. 目标用户与场景

医药/大健康市场部（品牌经理、市场专员、医学市场）。共同点：每周写医生材料，但没有"专门替你把关 message 的人"。

| 场景 | 输入 | 期望输出 |
|---|---|---|
| 写完一条 key message，没把握 | 一句定位/message | 评级 + 挑刺 + 落地版 |
| 要给医生写产品优势介绍 | 基础信息 | 体检 + 改写 + 成品文 |
| 想发小红书/公众号做个人 IP | 任一脱敏案例 | "改前 vs 改后"对比卡 |

## 5. 分层（将来的交易物形态）

- 免费：速检（扫大词虚词，出评级）——作为内容领取钩子。
- 低价：标准（+ 落地改写 + 成品文）。
- 进阶：深档（+ 模拟医生 Q&A + 对比卡 + 自定义词库）。

## 6. 刻意不做

- 不做"市场部全家桶"（BP/活动/汇报全包）——窄，才能被一个市场部人今天下午拿去用。
- 不做幻灯排版（红海，且非本引擎强项）。
- 不替用户做市场判断或编数据。
