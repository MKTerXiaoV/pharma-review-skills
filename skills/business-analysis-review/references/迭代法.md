# 越用越准:约束型迭代法（私域护城河）

> 业务分析体检是**约束型**(定义"一份分析读得够不够深/够不够全"的判断标准)。约束型最强的进化方式 = 把你真实改过的分析喂回来,让标准越来越贴你的判断。别人抄得走"四步五动作",抄不走"你读过几百份区域分析后的手感"。

## 四步循环（每审改一份真实分析就做一次）
1. **体检**一份真实数据/分析,我给挑刺 + 改写建议。
2. **你亲手改满意**(不是嘴上说,是真把它读深、补全)。
3. **让我提炼**:"你改了什么、背后是什么深度/覆盖标准?"——归纳成 ≤10 条规则。
4. **写回** `EXTEND.md` / `references/`(补进"你那条业务线该有的分析深度/必拉的源")。

## 注意
- 脱敏后再喂(去公司/产品/真实数据)。
- 规则 ≤10 条;长了让 AI 归纳合并,防打架。
- 几轮后,挑刺标准 = 你本人的分析水准。
