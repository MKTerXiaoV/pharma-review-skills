# 评审团协议(深档增强 · 显式点名才启动)

> 「评审团」= 把体检从"一个 AI 过一遍"升级成"**多位各守一个透镜的评委盲审 → 反驳者逐条对抗验证 → 主笔合成**"。
> 独立性是它的全部价值:评委互相看不见,反驳者只负责推翻。

## 何时用(闸门 · 先过这关)
- 只在用户**显式点名**(说"上评审团 / 评审团体检 / 多专家会诊")时启动,**绝不默认触发**。
- 启动前必告知成本:评审团会派出数十个子代理,消耗约为标准档的 **30–50 倍**;只值得用在正式汇报/过老板的完整分析报告上,日常看一个数、一段判断,速检/标准档足够。**按实际执行层级如实告知**——降级跑"多轮自审"时消耗远低于满配,别照念满配数字。
- **确认规则**:用户点名"上评审团"且分析已给全 = 视为确认,报一句成本即开跑,不再二次拦;点名但没给材料 → 先走首次引导收材料。

## 三级执行(按环境能力择一 · 诚实告知用户跑的是哪级)
1. **评审团**(首选):环境有多智能体编排工具(如 Claude Code 的 Workflow)→ 按本协议现场编排:评委并行盲审 → 合并去重 → 反驳者逐条投票 → 主笔合成。真并行、真盲审。
2. **并行子审**:环境只有并行子代理(如 Agent/Task 类工具)→ 每位评委/反驳者一个独立子代理,保住"盲审 + 独立反驳"实质;可把每条挑刺的反驳者从 4 位减到 2 位(**必留"行业惯例校准"**)。
3. **多轮自审**(兜底):单对话环境 → 逐透镜轮流批判(每轮只戴一个透镜,刻意不回看上一轮结论)→ 反驳轮 → 合成。**必须明确告知用户:这是评审团的单机版,后一轮看得见前一轮,独立性弱一档。** 单机版反驳轮可精简到 2 位(**必留"行业惯例校准"**);每轮重新读一遍该透镜的判据文件、禁止引用前轮条目编号——用行为约束近似盲审。

## 评委席(盲审 · 5 位 + 老板席)
| 评委 | 只看一件事 | 标准来源 |
|---|---|---|
| 覆盖与数据源 | 三视角(看自己/看周边/看全局)走全没?两关联(内外部/过程结果)补没?源拉全没(只拉内部结果数据=覆盖不及格)? | `覆盖与数据源.md` |
| 深度四步 | 解读→切割→关联→递进,到第几层?停在"谁大谁小"的单一描述=浅层 | `四步工序.md` |
| 5 习惯动作 | 看大小/看趋势/看变化Δ/看非常规/找标杆——做全没?描述完问没问原因? | `五动作与两关联.md` |
| 指标体系 | 份额/潜力/增长三维看了没?销潜比、漏斗用了没?有没有虚假精确、无出处的数? | `覆盖与数据源.md` + `不误读红线.md` |
| 导出行动 | 有没有 SWOT 导出策略、对象导向呈现?有结论没行动=半成品 | `导出行动.md` |

**⚠️ 一票否决标记**:任何评委发现「相关当因果 / 数据无出处·虚假精确 / 只描述结果数据没接外部」,在该条挑刺上标注【一票否决】——若该条通过反驳席存活,最终评级直接"未及格线,建议返工",不参与合议。

**老板席**(独立运行,不参与挑刺):扮演业务老板,给出汇报会上会当场顶回去的 **3 个最狠问题**(核心永远是"so what?要我批什么?":机会在哪/该投哪/下一步谁做什么),必须具体到这份分析。

每位评委规则:
- 只从自己的透镜挑刺,**1–4 条最狠的,宁缺毋滥**;每条必引原句/原数。
- 每条给:命中什么失败信号 → 为什么不够(覆盖/深度/动作) → 严重度(高/中/低) → 可落地的改写建议(补什么源/切什么维/递进到什么行动)。
- 各自独立给评级(及格/良好/优秀/未及格线)+ 一句话理由。评委之间互相看不见。

## 合并去重
指向同一问题的挑刺合并成一条(保留最狠表述 + 最好的改写建议),记录"哪些评委命中"——**多透镜共同命中本身是强度信号**。不丢独立问题、不自己新增。

## 反驳席(对抗验证 · 每条合并后挑刺过 4 位反驳者)
| 反驳者 | 攻击角度 |
|---|---|
| 业务老板视角 | 站在真实业务老板角度,这条挑刺影响决策吗?还是评委在秀方法论、老板根本不在乎? |
| 落地操盘手 | 做过上百份区域分析的老手:真问题还是吹毛求疵?建议补的源现实中拿得到吗(数据可得性/时效/成本)? |
| 方法论核查 | 有没有误用标准(如拿完整报告的五维标准轰用户随口问的一个数、把用户没给的数据当"没拉的源")?引用的原句/原数真的在输入里吗? |
| 行业惯例校准 | 这在真实医药市场分析里是不是**普遍且被接受**的做法?别把行业常态当硬伤(误读红线除外——相关当因果、编数据该拦必须拦)。 |

规则:每位反驳者的任务是**尽力推翻**这条挑刺,推翻不了才放行;模棱两可倾向推翻。**≥2 票推翻 = 毙掉**(进纪要);恰好 1 票反对 = 幸存,但反对意见作为"少数派意见"附在该条下。**例外:【一票否决】类挑刺(相关当因果/无出处/虚假精确)只核事实不做价值票决**——只要引用属实,直接保留;若定性本身有判断空间(如某表述算不算"虚假精确"),按普通挑刺走票决。

## 主笔合成(输出必须与标准档同构)
最终报告**落 SKILL.md 的统一输出模板五段式**(🔴卡在哪 → 🗣会被追问 → 🟢删/补/换 → 🎯证明 → 🚫失败信号),与速检/标准档同构、可截图沉淀。评级合议:取评委多数意见,平票取更保守(更低)的一档;**存活的【一票否决】条 = 直接未及格线**(不合议)。

评审团特有内容作为**追加段**,不另起格式:
```
⚖️ 评审团纪要(追加段)
- 各评委评级 + 分歧点
- 每条挑刺标注:✓N/4 票幸存 · 提出者透镜 · 少数派意见(如有)
- 被毙掉的挑刺 + 推翻理由(评审团的自我纠错,照实写,不藏)
- 老板席 3 问(并入 🗣 会被追问 段)
```

诚实红线:0 条被毙也照实说,并提醒用户**人工抽查 2–3 条**——反驳席不是万能的,行业判断的最后一道校准永远在人。
