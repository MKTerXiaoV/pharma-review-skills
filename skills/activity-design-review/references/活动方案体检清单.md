# 活动方案 · 体检自检清单

> 标准档逐项过;每个"否"= 一条挑刺。先做框架审计,再过 yes/no。

## 框架审计要点(先看大结构)
- 活动能**反推出年度策略**吗?(脱钩?)
- 活动**进化段位**:设计流程 → 设计体验 → 设计机制(停在哪一层?)
- **内核一(品牌利益点)+ 内核二(客户真实世界)**都兼顾了吗?
- **峰终定律**:峰值体验设计了吗?结束体验是什么?
- **主菜配菜比例**:核心医学内容占多少?(配菜喧宾夺主?)
- **客户参与度级别**:有趣 → 和我有关 → 专业升级 → 命运共同体(到第几级?)

## yes/no 自检(每个"否"挑出来)
- [ ] 医生参加完,**知道明天找什么特征的病人开始试**吗?(业务感)
- [ ] 日程有**节奏感**吗?(剥洋葱式递进 / 正-侧-反 多角度,而非平铺)
- [ ] 有没有**反人性设计**?(上传病例 / 现场录视频 / 填长问卷 → 有则砍)
- [ ] **峰值体验**是什么?**终值体验**是什么?(说不出 = 没设计)
- [ ] 活动在**积累什么资产**?(HCP 网络 / 代表能力 / 患者数据 —— 一次性消耗 vs 可复用沉淀)

## 输入信息不全时
方案没写受众/目标/日程细节 → 不脑补,标 `[待确认:受众未明]` 等,提示用户补,再给完整体检。
