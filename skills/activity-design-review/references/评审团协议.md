# 评审团协议(深档增强 · 显式点名才启动)

> 「评审团」= 把体检从"一个 AI 过一遍"升级成"**多位各守一个透镜的评委盲审 → 反驳者逐条对抗验证 → 主笔合成**"。
> 独立性是它的全部价值:评委互相看不见,反驳者只负责推翻。实测同一份方案,单次体检挑出 4 条,评审团挑出 16 条——多出的主要是合规和资产沉淀两条线(见 `examples/评审团-样张.md`)。

## 何时用(闸门 · 先过这关)
- 只在用户**显式点名**(说"上评审团 / 评审团体检 / 多专家会诊")时启动,**绝不默认触发**。
- 启动前必告知成本:评审团会派出数十个子代理,消耗约为标准档的 **30–50 倍**;只值得用在正式过评审/过老板、高价值方案上,日常体检标准档足够。**按实际执行层级如实告知**——降级跑"多轮自审"时消耗远低于满配,别照念满配数字。
- **确认规则**:用户点名"上评审团"且方案已给全 = 视为确认,报一句成本即开跑,不再二次拦;点名但没给方案 → 先走首次引导收方案。

## 三级执行(按环境能力择一 · 诚实告知用户跑的是哪级)
1. **评审团**(首选):环境有多智能体编排工具(如 Claude Code 的 Workflow)→ 按本协议现场编排:评委并行盲审 → 合并去重 → 反驳者逐条投票 → 主笔合成。真并行、真盲审。
2. **并行子审**:环境只有并行子代理(如 Agent/Task 类工具)→ 每位评委/反驳者一个独立子代理,保住"盲审 + 独立反驳"实质;可把每条挑刺的反驳者从 4 位减到 2 位(**必留"行业惯例校准"**)。
3. **多轮自审**(兜底):单对话环境 → 逐透镜轮流批判(每轮只戴一个透镜,刻意不回看上一轮结论)→ 反驳轮 → 合成。**必须明确告知用户:这是评审团的单机版,后一轮看得见前一轮,独立性弱一档。** 单机版反驳轮可精简到 2 位(**必留"行业惯例校准"**);每轮重新读一遍该透镜的判据文件、禁止引用前轮条目编号——用行为约束近似盲审。

## 评委席(盲审 · 7 位 + 老板席)
| 评委 | 只看一件事 | 标准来源 |
|---|---|---|
| 紧锁策略 | 能否反推出年度策略?目标是虚词还是具体的医生行为改变? | `五大虚症.md` 虚症① |
| 内核二·客户洞察 | 是"我方想讲什么"还是接住了医生的真实世界(为什么不用/卡在哪)? | `五大虚症.md` 虚症② |
| 体验净值·峰终·反人性 | 逐环节算 ∑正−∑负;有无要医生额外劳动的环节;峰值/终值在哪 | `体验公式与三技巧.md` |
| 业务感·主菜配菜 | 医生走出去知道"下一个病人怎么做"吗(没有=未及格线);主菜清晰吗、配菜有吗、参与度第几级 | `主菜配菜与参与度.md` |
| 系列化·资产沉淀 | 孤立活动还是可经营的系列/平台?留下什么可复用资产? | `体验公式与三技巧.md` 技巧三 |
| 合规红线 | 只挑真有暴露面的:夸大/超适应症隐患、无边界讨论、审核闭环缺失 | `合规红线.md` |
| 资深战略顾问 | 唯一标准"对生意有影响":开完会处方会变吗?变在哪类病人?不变这钱为什么花? | 通用外部视角 |

**老板席**(独立运行,不参与挑刺):扮演事业部老板,给出评审会上会当场顶回去的 **3 个最狠问题**(投入产出/行为改变/资产沉淀),必须具体到这份方案,不要通用问题。

每位评委规则:
- 只从自己的透镜挑刺,**1–4 条最狠的,宁缺毋滥**;每条必引方案原文/环节名。
- 每条给:命中什么虚症 → 为什么 → 严重度(高/中/低) → 可落地的改写建议。
- 先按 `类型与规模-体检侧重.md` 判规模(小会=个体说服/大会=群体观念塑造),别用小会标准卡大会。
- 各自独立给评级(及格/良好/优秀/未及格线)+ 一句话理由。评委之间互相看不见。

## 合并去重
指向同一问题的挑刺合并成一条(保留最狠表述 + 最好的改写建议),记录"哪些评委命中"——**多透镜共同命中本身是强度信号**。不丢独立问题、不自己新增。

## 反驳席(对抗验证 · 每条合并后挑刺过 4 位反驳者)
| 反驳者 | 攻击角度 |
|---|---|
| 参会医生视角 | 站在真实参会医生角度,这条挑刺成立吗?还是评委想多了、医生根本不在乎? |
| 落地操盘手 | 办过上百场会的老手:真问题还是吹毛求疵?改写建议在预算/执行/审批上落得了地吗? |
| 方法论核查 | 有没有误用标准(如用小会个体说服标准卡 80 人大会)?引用的原文真的在方案里吗? |
| 行业惯例校准 | 这在真实医药学术会议里是不是**普遍且被接受**的做法?别把行业常态当红线(例:脱敏病例的展示/讨论是学术会议常态,"有病例环节"本身不是隐私红线;真正的暴露面只有:即兴提交难保证脱敏质量 / 企业留存要有去向 / 征集不得挂钩报酬)。 |

规则:每位反驳者的任务是**尽力推翻**这条挑刺,推翻不了才放行;模棱两可倾向推翻。**≥2 票推翻 = 毙掉**(进纪要);恰好 1 票反对 = 幸存,但反对意见作为"少数派意见"附在该条下(少数派意见常常救了改写方案——照实附)。

## 主笔合成(输出必须与标准档同构)
最终报告**落 SKILL.md 的统一输出模板五段式**(🔴卡在哪 → 🗣会被追问 → 🟢删/补/换 → 🎯证明 → 🚫失败信号),与速检/标准档同构、可截图沉淀。评级合议:取评委多数意见,平票取更保守(更低)的一档;业务感缺失仍是未及格线硬标准。

评审团特有内容作为**追加段**,不另起格式:
```
⚖️ 评审团纪要(追加段)
- 各评委评级 + 分歧点
- 每条挑刺标注:✓N/4 票幸存 · 提出者透镜 · 少数派意见(如有)
- 被毙掉的挑刺 + 推翻理由(评审团的自我纠错,照实写,不藏)
- 老板席 3 问(并入 🗣 会被追问 段)
```

诚实红线:0 条被毙也照实说,并提醒用户**人工抽查 2–3 条**——反驳席不是万能的,行业判断的最后一道校准永远在人。
