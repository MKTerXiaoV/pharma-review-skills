# 测试用例(活动设计体检 v0)· 脱敏

> 发布前必跑。每条 = 输入 + 客观验收点(skill 该挑出什么/给什么评级)。有 skill vs 无 skill 对比,看是否真有增益。改版后回归。

## T1 · 典型(策略脱钩 + 反人性 + 没业务感)
**输入**:见 `examples/活动方案-样张.md` 的方案。
**验收点**:评级=未及格线(建议返工);必须挑出 ① 目标策略脱钩 ② 现场上传病例=反人性 ③ 结尾致辞=没业务感;改写须给"明天这样的病人这样试"类动作。

## T2 · 边界:已经很好的方案
**输入**:见 `tests/T2-良好方案.md`(紧锁策略、有内核二、无反人性、结尾发过审 next-step 卡片、有峰值、系列化)。
**验收点**:评级=良好/优秀;不硬挑刺;指出还能提升的 1 点(如参与度升到"命运共同体"/资产沉淀),不编问题;**不得把已做合规前置的病例环节挑成隐私红线**(行业常态)。

## T3 · 边界:全是虚症
**输入**:纯吃喝+抽奖+大咖合影、零医学主菜的"活动"。
**验收点**:评级=未及格线(建议返工);一票否决(无业务感);点出主菜缺失,但**不替用户重设计活动**(守"只体检不代设计")。

## T4 · 边界:信息不全
**输入**:只有一句"我要办个论坛,帮我看看"。
**验收点**:不脑补;走速检/首次引导,问清受众/目标/日程再体检;标 `[待确认]`。

## T5 · 合规
**输入**:方案宣传语含"根治高血压""唯一推荐",有与处方挂钩的礼品。
**验收点**:挑出夸大/绝对化 + 诱导性福利(红线);结尾 MA-RA 提示。

## T6 · 评审团档回归(需多智能体编排环境;无则按协议降级验证)
**输入**:`tests/T2-良好方案.md` 的方案 + 用户点名"上评审团"。
**验收点**:① 启动前先报成本(约标准档 30–50 倍)并等用户确认;② 评级=良好/优秀,**不为产出硬挑刺**(不得虚构"高"严重度问题);③ 反驳席含"行业惯例校准",有实际推翻/降级记录或如实报告 0 条被毙;④ 输出落统一契约五段式 + ⚖️ 评审团纪要追加段;⑤ 无编排环境时降级为"多轮自审"并向用户明说独立性弱一档。

## 跑法
- 装 skill,逐条把输入发给 agent,对照验收点。
- 无 skill 跑同一输入对比:无 skill 易漏"反人性/业务感/合规"——证明增益。
- 改 SKILL/references 后回归这 5 条。
