# 测试用例(活动设计体检 v0)· 脱敏

> 发布前必跑。每条 = 输入 + 客观验收点(skill 该挑出什么/给什么评级)。有 skill vs 无 skill 对比,看是否真有增益。改版后回归。

## T1 · 典型(策略脱钩 + 反人性 + 没业务感)
**输入**:见 `examples/活动方案-样张.md` 的方案。
**验收点**:评级=未及格线(建议返工);必须挑出 ① 目标策略脱钩 ② 现场上传病例=反人性 ③ 结尾致辞=没业务感;改写须给"明天这样的病人这样试"类动作。

## T2 · 边界:已经很好的方案
**输入**:一个紧锁策略、有内核二、无反人性、结尾发 next-step 卡片、有峰值的方案。
**验收点**:评级=良好/优秀;不硬挑刺;指出还能提升的 1 点(如参与度升到"命运共同体"/资产沉淀),不编问题。

## T3 · 边界:全是虚症
**输入**:纯吃喝+抽奖+大咖合影、零医学主菜的"活动"。
**验收点**:评级=未及格线(建议返工);一票否决(无业务感);点出主菜缺失,但**不替用户重设计活动**(守"只体检不代设计")。

## T4 · 边界:信息不全
**输入**:只有一句"我要办个论坛,帮我看看"。
**验收点**:不脑补;走速检/首次引导,问清受众/目标/日程再体检;标 `[待确认]`。

## T5 · 合规
**输入**:方案宣传语含"根治高血压""唯一推荐",有与处方挂钩的礼品。
**验收点**:挑出夸大/绝对化 + 诱导性福利(红线);结尾 MA-RA 提示。

## 跑法
- 装 skill,逐条把输入发给 agent,对照验收点。
- 无 skill 跑同一输入对比:无 skill 易漏"反人性/业务感/合规"——证明增益。
- 改 SKILL/references 后回归这 5 条。
