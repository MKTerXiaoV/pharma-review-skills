# 样张:活动设计体检(虚构 · 某降压药城市论坛)

> 脱敏虚构例,演示 input → 体检 → 改后全程。配图见 `活动方案-对比卡.png`。

## 输入(用户给的方案)
> **活动**:某降压药城市心血管论坛 · 受众:三级医院心内科医生 80 人
> **日程**:09:00 开幕致辞 → 上午 4 场专家授课(指南解读/机制/循证/安全性)→ 午餐 → 14:00 圆桌讨论 → **现场请每位医生上传 1 例自己的病例** → 17:00 主席总结致辞、合影。
> **目标**:扩大产品学术影响力。

## 体检输出(标准档)

**评级:未及格线,建议返工** — 议程热闹,但反推不出策略,有反人性设计,结尾没有业务感。

🔴 **挑刺(逐条引原文)**
1. 〔目标〕"扩大学术影响力" → **策略脱钩**:反推不出年度策略。这论坛要让医生改变什么具体行为?(命中虚症1)
2. 〔上午 4 场授课〕**主菜配菜失衡 + 内核二缺失**:全是"我方内容"(指南/机制/循证),没有一场接"医生为什么晨峰管理做不好"的真实困境。(虚症2)
3. 〔现场上传病例〕**反人性设计**:要医生当场额外劳动 → 砍。(虚症3)
4. 〔17:00 主席总结致辞、合影〕**没有峰值、没有业务感**:平铺收尾,医生走出去不知道明天干啥。(虚症4+5)

🟢 **改写建议**
- 目标改成可反推策略:"让医生对**晨峰控制不佳**的患者,优先想到本方案"。
- 上午留 1 场讲"晨峰漏管的真实病人"(内核二);授课压到 2–3 场(主菜聚焦)。
- 取消病例上传,改"主办方现场示范 1 例晨峰管理",医生看不干活。
- 设峰值:一个"晨峰未控 → 心血管事件"的震撼真实数据时刻,绑核心信息。
- 结尾发**「晨峰 3 类病人 next-step 卡片」**(业务感):医生带走"明天这样的病人这样试"。

🎯 **深档(模拟老板提问)**
- "这 80 个医生开完会,下周处方会变吗?变在哪类病人?"
- "这场花的钱,留下了什么可复用资产?"(HCP 网络?可二次传播内容?)

收尾:本体检为方法校准,具体活动材料须经 MA/RA 审核后执行。
