# PRODUCT · 活动设计体检(为什么)

## 它解决什么
医药市场部天天办活动/论坛/科室会,最常见的病不是"不会办",是**办得热闹但推不出策略、医生走出去不知道明天怎么做**。市面上没人帮你"审"一个活动方案对不对——大家都在教你"怎么设计活动"(生成,红海)。

## 它的不一样
- **审视,不生成**:不替你设计,帮你**挑刺 + 校准判断**(护城河 = 判断)。
- **有招牌一刀**:"业务感"——医生参加完知道"下一个病人这样试"吗?这是一线带过队的人才有的标准。
- **独门壁垒**:整套判断建在一线市场人的真实方法上(五大虚症 + 体验公式 + 三技巧),"业务感/体验净值"这类标准别人编不出。

## 目标用户
医药市场部(产品经理/活动负责人)、医学市场、想让活动"有业务感"的人。

## 母题
市场部判断力如何被 Agent 流程化 —— 这是"活动设计判断"的产品化。判断流水线里的一道工序(简报 → 关键信息体检 → 活动设计体检)。

## 形态
开源 skill(Claude Code / Codex / OpenClaw 可装)。轻端钩子;团队版字段(复核人/风险边界/省时)= 重端筛子。
